L'autorisation est une partie clé de toute solution construite avec Codat. Chacun de vos clients PME doit autoriser l'accès à ses données avant que vous puissiez utiliser ces données. Un flux d'autorisation fluide et rassurant est essentiel pour recevoir l'autorisation de vos clients PME.

Pour y parvenir, utilisez notre [composant Link SDK](/auth-flow/authorize-embedded-link) et intégrez notre flux d'autorisation Link dans votre application.

![](/img/auth-flow/embedded-link-selection.png)

## Qu'est-ce que Link ?

Link est notre parcours d'autorisation en marque blanche pré-construit, conçu en utilisant notre vaste expérience de construction et les meilleures pratiques d'autorisation pour maximiser la conversion.

Intégrez ce flux d'autorisation de premier ordre dans votre parcours utilisateur avec notre [composant Link SDK](/auth-flow/authorize-embedded-link) et profitez d'un **taux de complétion moyen du flux d'autorisation de 89%**.

Avec nos [exemples riches](/auth-flow/authorize-embedded-link#get-started) et une <a href="https://links.codat.io/client/873ff19e-6fe0-47b0-a4e1-e19f344c78f6?user=8ee6c557-949c-40a8-b31d-e1fa02ef7fbc" target="_blank">démo interactive</a>, nous avons facilité votre démarrage. Vous pouvez avoir le flux en cours d'exécution dans votre code frontend avec seulement quelques lignes de code.

### Fonctionnalités

- **Interface utilisateur intuitive** basée sur notre expertise et les meilleures pratiques apprises garantit un flux d'autorisation à haute conversion
- **Authentification** conforme aux normes OAuth 2.0
- **Interface utilisateur personnalisable** qui reflète la marque de votre entreprise
- **Compatible React et non-React** composant JavaScript pré-construit pour une implémentation rapide
- **Importations dynamiques** signifient que votre flux d'autorisation ne prendra jamais de retard sur notre API

:::tip SDK de gestion des connexions

Améliorez la fonctionnalité du flux d'autorisation et donnez à vos clients la possibilité de gérer les autorisations d'accès en utilisant notre [SDK de gestion des connexions](/auth-flow/optimize/connection-management) dans votre code frontend.

:::

## Solutions prises en charge

Link est compatible avec toute notre [gamme de solutions](/using-the-api/overview#apis).

## Intégrations prises en charge

La compatibilité du navigateur et du mobile de Link varie selon les différentes intégrations :

- Certaines intégrations sont **uniquement sur ordinateur de bureau**, par exemple [Sage 50](/integrations/accounting/sage50/accounting-sage50), [QuickBooks Desktop](/integrations/accounting/quickbooksdesktop/accounting-quickbooksdesktop).
- D'autres nécessitent **le téléchargement d'un package de bureau**, par exemple [Oracle NetSuite](/integrations/accounting/netsuite/accounting-netsuite), [Microsoft Dynamics 365 Business Central](/integrations/accounting/dynamics365businesscentral/accounting-dynamics365businesscentral).
- D'autres intégrations nécessitent de passer à des flux complexes sur **des sites web non optimisés pour le mobile** en raison de la complexité de la plateforme tierce, par exemple [Sage 200 Standard](/integrations/accounting/sage200/accounting-sage200), [Sage Intacct](/integrations/accounting/sage-intacct/accounting-sage-intacct).

:::tip Gestion du changement

Comme toutes les solutions Codat, Link SDK est soumis à [notre politique de gestion du changement](/using-the-api/change-policy). Nous donnerons un préavis approprié pour les modifications de notre interface utilisateur de flux d'autorisation et de toutes les API associées. Nous avons des mesures rigoureuses de test et de sécurité en place pour vous assurer que vous pouvez importer notre SDK en toute confiance.

Nous fournissons également des mises à jour dans notre [journal des modifications du SDK](/auth-flow/authorize-embedded-link#changelog).

:::
